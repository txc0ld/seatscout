# Security Policy

## Supported Surface

SeatScout is currently a static frontend application with a curated local dataset. The main security surfaces are:

- dependency integrity
- build and workflow configuration
- any future user-input or network integrations

## Reporting a Vulnerability

Please do **not** open a public issue for a suspected vulnerability.

Preferred disclosure paths:

1. Open a private GitHub security advisory for this repository.
2. If private advisory access is unavailable, contact the repository owner directly through GitHub.

Include:

- a short description of the issue
- affected files or flows
- reproduction steps
- impact assessment
- any suggested remediation

## Response Expectations

- Acknowledge receipt as soon as practical
- Reproduce and assess severity
- Ship the smallest safe fix
- Publish a coordinated note only after the fix is available if disclosure is appropriate
